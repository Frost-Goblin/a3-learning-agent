# 综合案例：学生成绩管理系统

## 项目定位

学生成绩管理系统是 Python 入门阶段非常适合的综合案例。它覆盖输入输出、分支循环、函数拆分、数据结构、文件处理等核心知识点，能够检验学生是否真正具备“从零散语法到完整程序”的迁移能力。

## 项目目标

实现一个命令行版学生成绩管理系统，至少支持以下功能：

1. 添加学生信息
2. 查询学生信息
3. 修改学生成绩
4. 删除学生记录
5. 显示全部学生信息
6. 统计平均分、最高分、最低分
7. 将数据保存到文件

## 建议数据结构

推荐使用“列表中存字典”的方式。

```python
students = [
    {"id": "2026001", "name": "Alice", "score": 91},
    {"id": "2026002", "name": "Bob", "score": 84}
]
```

## 模块拆分建议

### 主程序模块

- 显示菜单
- 接收用户输入
- 调度功能函数

### 数据处理模块

- 添加
- 删除
- 修改
- 查询

### 统计分析模块

- 平均分
- 最高分
- 最低分
- 及格率

### 文件模块

- 保存数据
- 读取数据

## 开发步骤建议

1. 先写内存版，不保存文件。
2. 再补查询、修改、删除。
3. 再加统计功能。
4. 最后接文件存储和异常处理。

## 典型难点

- 不会设计统一的数据结构
- 菜单循环和功能函数耦合太紧
- 删除或修改时找不到目标记录
- 文件存储格式不统一

## 评分关注点

- 是否具备完整闭环
- 是否使用函数拆分
- 是否有合理的数据结构
- 是否有异常处理
- 是否有基本可维护性

## 可扩展方向

- 支持多门课程成绩
- 支持排序与筛选
- 支持导出为 CSV
- 支持班级维度统计
