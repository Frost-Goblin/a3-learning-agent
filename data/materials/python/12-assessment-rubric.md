# Python 课程考核标准参考

## 日常作业关注点

1. 能否正确运行。
2. 是否完成题目要求。
3. 命名是否基本清楚。
4. 是否有最基本的结构拆分。

## 实验任务关注点

1. 是否掌握本实验对应的核心知识点。
2. 是否能解释自己为什么这样写。
3. 是否能指出运行中的典型错误。
4. 是否能独立完成基础部分，扩展部分完成多少。

## 综合项目关注点

1. 功能是否闭环。
2. 数据结构是否合理。
3. 是否用函数拆分主要功能。
4. 是否有简单异常处理。
5. 是否便于后续继续扩展。

## 常见扣分点

- 程序能跑，但所有逻辑都塞在主流程。
- 功能做出来了，但结构混乱，后面很难改。
- 能抄出结果，但无法解释思路。
- 报错以后不会定位问题来源。

## 适用场景

- 生成作业评价
- 生成项目点评
- 生成改进建议
