# A3 项目设计框架目标

## 1. 项目定位

本项目面向第十五届中国软件杯 A3 赛题，选择《Python 程序设计》作为首期课程，构建一个个性化学习资源生成与学习多智能体系统。

系统不做泛化的大而全教育平台，而是聚焦一门课、一个学习闭环、一个可演示原型，重点体现多智能体协同、学生画像、个性化资源生成和动态学习路径规划。

## 2. 核心目标

系统目标是让学生通过自然语言对话完成学习画像构建，再由多个智能体协作生成适配其基础、偏好和薄弱点的学习资源，并根据练习与反馈持续调整后续学习方案。

核心闭环：

```text
对话建档 -> 知识检索 -> 资源生成 -> 路径规划 -> 测评反馈 -> 画像更新
```

## 3. 课程边界

首期课程固定为《Python 程序设计》。

课程知识点包括：

- 变量与数据类型
- 控制流
- 函数与模块
- 文件处理
- 面向对象
- 异常与调试

选择 Python 的原因：

- 生成讲解、题库、代码案例都比较稳定
- 可演示性强，适合展示个性化学习效果
- 对比赛文档、PPT 和演示视频都友好
- 后续可扩展到人工智能导论、数据结构等课程

## 4. 智能体划分

### 4.1 学生画像 Agent

职责：

- 通过对话收集学生基础、学习目标、学习历史和偏好
- 抽取不少于 6 个画像维度
- 根据后续学习行为动态更新画像

画像维度：

- 知识基础
- 学习节奏
- 认知偏好
- 易错点或薄弱点
- 学习动机
- 评估反馈偏好

### 4.2 知识检索 Agent

职责：

- 管理课程资料、章节知识点、例题和案例
- 根据学生问题和学习目标检索相关内容
- 为资源生成提供依据，降低大模型幻觉风险

### 4.3 资源生成 Agent

职责：

- 根据学生画像和课程知识点生成个性化学习资源
- 输出至少 5 类资源

资源类型：

- 课程讲解文档
- 知识点思维导图
- 分层练习题库
- 拓展阅读材料
- 代码实操案例

### 4.4 路径规划 Agent

职责：

- 将生成资源组织为阶段化学习计划
- 根据学生基础和学习节奏安排学习顺序
- 输出按天或按周的学习路径

路径阶段：

- 画像建档
- 知识补齐
- 任务训练
- 复盘评估

### 4.5 效果评估 Agent

职责：

- 汇总答题结果、资源使用情况和反馈数据
- 生成学习效果报告
- 给出下一轮资源推荐和路径调整建议

## 5. 功能模块

### 学生端

- 对话式画像构建
- 课程知识点选择
- 个性化资源生成
- 学习路径查看
- 练习与评估反馈

### 管理端

- 课程资料管理
- 知识点管理
- 题库与案例管理
- 学生画像查看
- 生成记录与评估报告查看

### 演示端

- 多智能体协同流程展示
- 生成轨迹展示
- 资源卡片展示
- 学习路径与评估面板展示

## 6. 技术路线

前端：

- React
- TypeScript
- Vite

后端候选：

- FastAPI
- Spring Boot

知识库：

- 课程文档解析
- RAG 检索增强
- 向量数据库可选 Chroma、FAISS 或 Milvus

模型：

- 优先考虑讯飞星火 API
- 也可预留通义千问、DeepSeek、豆包等模型接口

数据：

- 学生画像表
- 课程知识点表
- 资源生成记录表
- 题库与作答记录表
- 学习路径与反馈表

## 7. MVP 目标

第一阶段只做可演示 MVP。

必须完成：

- 可运行 Web 原型
- 学生画像配置与展示
- 5 类资源生成结果展示
- 个性化学习路径展示
- 学习效果评估展示
- 多智能体流程说明

暂不强求：

- 真实登录系统
- 完整后台权限
- 大规模真实课程库
- 复杂模型微调
- 移动端原生应用

## 8. 作品表达重点

方案和文档中重点强调：

- 系统不是简单聊天机器人，而是多智能体协作系统
- 学生画像会影响资源生成和路径规划
- 知识库检索用于约束生成内容，降低幻觉
- 学习评估结果会反向更新画像和推荐策略
- 系统可以从 Python 扩展到其他高校课程
